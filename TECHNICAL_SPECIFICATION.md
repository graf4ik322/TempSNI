# ТЕХНИЧЕСКОЕ ЗАДАНИЕ
## Веб-приложение TwchPlaceholder

### 1. ОБЩЕЕ ОПИСАНИЕ ПРОЕКТА

**Цель проекта:** Разработка веб-приложения для замещения контента (placeholder) с возможностью интеграции со сторонними сервисами без использования собственного бекенда.

**Тип решения:** Frontend-only приложение на чистых веб-технологиях

**Ограничения:**
- ❌ Нет собственного бекенда
- ❌ Нет API endpoints
- ❌ Нет CORS прокси
- ❌ Нет внешних API запросов
- ❌ Нет фреймворков (React, Vue, Angular)
- ✅ Только vanilla HTML/CSS/JavaScript
- ✅ Работа через iframe для внешнего контента
- ✅ Локальное хранение данных

### 3. ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ

#### 3.1 Основные функции

**F1. Отображение placeholder контента**
- Показ заглушки с настраиваемым содержимым
- Поддержка различных типов контента (текст, изображения, iframe)
- Адаптивный дизайн для всех устройств

**F2. Настройка placeholder**
- Изменение текста заглушки через input поля
- Загрузка пользовательских изображений (через File API и drag&drop)
- Ввод URL для iframe контента
- Выбор из предустановленных шаблонов
- Настройка цветовой схемы через color picker

**F3. Работа с внешним контентом**
- Отображение внешних сайтов через iframe
- Проверка URL на валидность
- Обработка ошибок загрузки iframe

#### 3.2 Дополнительные функции

**F4. Пользовательский интерфейс**
- Темная/светлая тема
- Полноэкранный режим
- Drag & Drop для загрузки файлов

**F5. Экспорт настроек**
- Сохранение конфигурации в JSON файл (через Blob API)
- Импорт настроек из файла (через FileReader API)
- Шаринг настроек через URL (URL parameters или hash)

### 4. ПОЛЬЗОВАТЕЛЬСКИЕ СЦЕНАРИИ

#### UC1. Создание простого placeholder
1. Пользователь открывает приложение
2. Выбирает тип контента (текст/изображение)
3. Настраивает содержимое
4. Сохраняет настройки локально
5. Placeholder отображается в полноэкранном режиме

#### UC2. Добавление iframe контента
1. Пользователь вводит URL внешнего сайта
2. Система проверяет валидность URL
3. URL сохраняется в настройках
4. Placeholder отображает контент через iframe

#### UC3. Шаринг конфигурации
1. Пользователь создает настройки
2. Экспортирует в JSON или получает URL
3. Другой пользователь импортирует настройки
4. Placeholder воссоздается с теми же параметрами

### 6. ТЕХНИЧЕСКИЕ ОГРАНИЧЕНИЯ И РЕШЕНИЯ

#### 6.1 Работа без бекенда

**Проблема:** Нет серверной части для обработки данных
**Решение:** 
- Использование публичных API с JSONP или поддержкой CORS
- Локальное хранение всех пользовательских данных
- Client-side маршрутизация

**Проблема:** Ограничения внешнего контента
**Решение:**
- Использование iframe для отображения внешних сайтов
- Предупреждения о X-Frame-Options ограничениях
- Fallback на сообщение об ошибке при блокировке iframe

#### 6.2 Производительность

**Оптимизация:**
- Lazy loading компонентов
- Debounce для пользовательского ввода
- Service Worker для кэширования
- Минификация и сжатие ресурсов
